ePO Setup

Overview

Trellix ePO (ePolicy Orchestrator) provides centralized security management for your entire enterprise security ecosystem. This guide covers the complete setup and configuration process for enterprise environments.

Trellix ePolicy Orchestrator (ePO)
Latest Version: 5.10.0 Part of Trellix XDR
Official Documentation

System Requirements

  • Operating System - Windows Server 2016/2019/2022
  • Database - Microsoft SQL Server 2016/2019/2022
  • Web Browser - Chrome, Firefox, or Edge (latest versions)

Database Configuration

Advanced 45 minutes

Database Overview

The Trellix platform requires a properly configured Microsoft SQL Server database for optimal performance and data storage. This guide covers setting up SQL Server for Trellix, configuring replication, and optimizing performance for enterprise deployments.

Microsoft SQL Server for Trellix
Recommended: SQL Server 2019 Critical for Trellix Platform
SQL Server Documentation

Pre-requisites

  • Microsoft SQL Server 2019+ - Standard or Enterprise edition
  • Minimum 16GB RAM allocated to database services
  • 100GB+ storage with high I/O performance (SSD recommended)
  • Network connectivity between Trellix server and database
  • SQL Server Authentication - Mixed Mode authentication enabled

SQL Server Configuration

Select your database deployment type:
Standard Deployment

Single SQL Server instance for small to medium environments

  • Simplified management
  • Lower hardware requirements
  • Easier backup and recovery
Enterprise Deployment

High-availability SQL cluster for large environments

  • Fault tolerance and failover
  • Enhanced performance
  • Better scalability
Standard SQL Server Setup
Single-Instance
45 minutes Complexity: Medium
Install Microsoft SQL Server

Install SQL Server with Mixed Mode authentication enabled. Ensure TCP/IP protocol is enabled in SQL Server Configuration Manager.

Create Trellix Database
CREATE DATABASE TrellixEPO COLLATE SQL_Latin1_General_CP1_CI_AS;

Set the database collation to SQL_Latin1_General_CP1_CI_AS for compatibility with Trellix components.

Configure Database User
CREATE LOGIN TrellixUser WITH PASSWORD = 'StrongPassword123!'; USE TrellixEPO; CREATE USER TrellixUser FOR LOGIN TrellixUser; EXEC sp_addrolemember 'db_owner', 'TrellixUser';

Create a dedicated database user with appropriate permissions for the Trellix application.

Configure SQL Server Properties

Set SQL Server memory limits, recovery model, and auto-growth settings based on your environment scale.

-- Set memory limits EXEC sp_configure 'show advanced options', 1; RECONFIGURE; EXEC sp_configure 'max server memory (MB)', 12288; -- 12GB for SQL Server RECONFIGURE;
Configure Maintenance Plans

Set up SQL Server maintenance plans for index maintenance, statistics updates, and backups.

Enterprise SQL Server Setup
High-Availability Cluster
120 minutes Complexity: High
Install SQL Server on Multiple Nodes

Install Microsoft SQL Server on all cluster nodes with the same configuration and patch level.

Configure Windows Server Failover Clustering

Set up Windows Server Failover Clustering (WSFC) as the foundation for SQL Server Always On Availability Groups.

Set Up Always On Availability Groups
-- Enable Always On feature EXEC sp_configure 'show advanced options', 1; RECONFIGURE; EXEC sp_configure 'hadr enabled', 1; RECONFIGURE;

Configure SQL Server Always On Availability Groups for high availability and disaster recovery.

Create Trellix Database with Replication
CREATE DATABASE TrellixEPO COLLATE SQL_Latin1_General_CP1_CI_AS; -- Add to Availability Group ALTER AVAILABILITY GROUP [AG_Trellix] ADD DATABASE [TrellixEPO];

Create the Trellix database and add it to the availability group for replication.

Configure Database User with Replication
-- Create login on all instances CREATE LOGIN TrellixUser WITH PASSWORD = 'StrongPassword123!'; -- Create user in the database USE TrellixEPO; CREATE USER TrellixUser FOR LOGIN TrellixUser; EXEC sp_addrolemember 'db_owner', 'TrellixUser';

Create users on all SQL instances and ensure proper synchronization.

Configure Advanced SQL Server Settings

Optimize SQL Server settings for enterprise performance including memory allocation, TempDB configuration, and more.

Database Maintenance

Regular maintenance is crucial for optimal Microsoft SQL Server performance. Set up the following maintenance tasks:

Index Maintenance

Schedule weekly index reorganization or rebuilds to maintain query performance.

-- Sample index maintenance script DECLARE @TableName NVARCHAR(255) DECLARE TableCursor CURSOR FOR SELECT table_name FROM information_schema.tables WHERE table_type = 'base table' OPEN TableCursor FETCH NEXT FROM TableCursor INTO @TableName WHILE @@FETCH_STATUS = 0 BEGIN EXEC('ALTER INDEX ALL ON ' + @TableName + ' REBUILD') FETCH NEXT FROM TableCursor INTO @TableName END CLOSE TableCursor DEALLOCATE TableCursor
Statistics Updates

Update statistics daily to ensure the SQL Server query optimizer has current information.

-- Update statistics for all tables EXEC sp_updatestats;
Backup Strategy

Implement SQL Server full backups weekly, differential backups daily, and transaction log backups hourly for comprehensive recovery capabilities.

-- Full backup BACKUP DATABASE TrellixEPO TO DISK = 'D:\Backups\TrellixEPO_Full.bak'; -- Differential backup BACKUP DATABASE TrellixEPO TO DISK = 'D:\Backups\TrellixEPO_Diff.bak' WITH DIFFERENTIAL; -- Transaction log backup BACKUP LOG TrellixEPO TO DISK = 'D:\Backups\TrellixEPO_Log.trn';

Performance Optimization

For optimal Microsoft SQL Server performance with Trellix products:

  • Properly size data and log files - Pre-allocate files to their expected size to reduce auto-growth events
  • Use multiple TempDB files - Configure one TempDB file per CPU core (up to 8) to reduce contention
  • Place files on separate volumes - Put data files, log files, and TempDB on separate physical storage
  • Ensure proper indexing - Monitor query performance and add indexes to frequently queried columns
  • Regular maintenance - Follow the maintenance plan outlined above
Microsoft SQL Server Best Practice

For large Trellix deployments (10,000+ endpoints), consider implementing SQL Server Resource Governor to protect critical database resources and ensure consistent performance during peak load periods.

Trellix HX Overview

Trellix Host Endpoint Security (HX) provides advanced endpoint detection and response capabilities with real-time monitoring, threat intelligence integration, and automated containment options. This guide covers installation, configuration, and best practices for enterprise deployments.

Trellix Host Endpoint Security (HX)

Essential Commands

This section provides a comprehensive reference for commonly used Trellix command-line tools and their applications in administration, troubleshooting, and maintenance tasks.

Server Management Commands

# Check status of ePO services
sudo service eposerver status

# Restart ePO server
sudo service eposerver restart

# Start all Trellix services
sudo trellix-service --start-all

# Stop all Trellix services
sudo trellix-service --stop-all

# Get service information
sudo trellix-service --info

Basic service management commands for Trellix servers

Configuration Commands

# Configure database connection
trellix-config --set-db-credentials --username [USER] --password [PASS]

# Configure server properties
trellix-config --set-property SERVER_NAME=trellix-primary

# Export current configuration
trellix-export --output config.xml --include-credentials

# Import configuration
trellix-import --input config.xml --override

# Verify system integrity
trellix-verify --check-all

Configuration management commands

Agent Management Commands

Agent Deployment
trellix-deploy --target 192.168.1.0/24 --credentials domain/admin

Deploy Trellix agents to all systems in the specified subnet using provided credentials.

Agent Status
trellix-agent-status --list-all --format=csv > agent_status.csv

Export the status of all agents to a CSV file for review and analysis.

Wake Up Agents
trellix-agent-wakeup --group "Finance Dept" --action scan

Trigger immediate check-in and scan for all agents in the Finance Department group.
